## Overview

This document identifies existing client projects that could be converted into case studies for Week 5-6. The goal is to find work already completed that demonstrates accessibility, AI integration, or CMS capabilities.

---

## Potential Case Study Candidates

### Assessment Criteria

For a project to be a good case study candidate, it should have:

1. **Measurable Results**
   - Before/after metrics available
   - Clear improvement demonstrated
   - Quantifiable outcomes

2. **Relevant to Government Contracts**
   - Accessibility work (WCAG compliance)
   - AI/automation implementation
   - CMS/system improvements
   - Process optimization

3. **Client Permission Likely**
   - Good relationship with client
   - Client values the work done
   - Client may benefit from case study exposure

4. **Documentation Available**
   - Project details documented
   - Results/metrics recorded
   - Screenshots or demos available
